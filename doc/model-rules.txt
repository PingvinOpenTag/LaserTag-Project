При большом количестве оборудования и великом множестве
было бы полезно унифицировать интерфейс модели.
Это позволит запускать любую модель на любой железячки.

В настоящем документа описаны основные принципы взаимодействия
между firmware (часть которая работает с железом) и моделью.

Модель представляет собой набор функций. Функции
делятся на два вида:
* действие,
* реакция.

Действия, это функции-обработчики событий, к примеру firmware
получила информацию о том что "словила" сигнал, она передаёт этот
сигнал модели (вызывает функцию из класса "действие"). После 
обработки со стороны модели выяснилось, что необходимо перейти
в режим "смерти" игрока, вызывается функция из класса реакция.

== Действия ==

* E_init() -- питание подано, firmware инициализировалась.
  После этого вызывается данная функция, которая даёт возможность
  инициализироваться модели.
* E_hitting(int group, int pid, int power, int loc) -- произошло
  попадание в игрока. Передаются уже разобранная информация.
* E_shoot() -- нажата кнопка выстрела.

== Реакция ==

* H_sendcode(int id, int type, int code)  -- Отправить команду. 
  В качестве аргументов номер устройства передачи, тип передачи,
  сам код.
* H_setstatus() -- устанавливает статус. Например: жив, мёртв.
* H_empty_gun() -- указывает firmware на то, что пистолет пуст.
