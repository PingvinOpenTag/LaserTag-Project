Правила работы с git репозиторием.

В идеальном случае полезно знать, как пользоваться git-ом,
представлять его основную идеалогию и систему работы.

В текущим документе представлена очень краткая "выжимка" из
документации, которая позволит:
* Скачать проект
* Закачать все модули проекта
* Провести изменения
* Добавить изменения в git
* Придерживаться правил ведения версий 

== Несколько слов о репозитори==

Git -- это программа ведения верйсий, разработанная 
тов. Линусом, для удобства разработки linux(ядро).
Система оказалась очень удачной и является хорошей
заменой заменой Subversion.

В отличии от subversion, git является децентрализованной 
системой ведения версий, это означает что каждая копия
репозитория, это отдельный автономный репозиторй, в который
можно вносить изменения и тп.

Для совместной работы выбирается главный репозиторий,
куда вносятся правки. 

Главный репозиторий находится на хостинге github.
github одна из самых популярных систем для проектов
в системе git.

== Скачивание репозитория ==

Репозитории можно скачивать как только для чтения
(доступ имеют все), либо для чтения/записи.
Для получения доступа на запись необходимо написать 
администрации с просьбой, включить его в проект.
Перед этим необходимо пройти процецеруду регистрации на
github.com.

В случае если по каким-то причинам получить доступ на запись
не получается, можно сделать форк проекта и создавать
"запросы на изменение".

Для скачивания репозитория необходимо набрать комаду:
* FIXME

== Работа с репозитория ==

При работе с депозитарием необходимо ознакомится с 
правилами разработки на основе git flow.

Основные причины использования. Необходимо максимально
упростить жизнь пользователям, т.е. тем кто слабо представляет 
себе что это за проект. Например, в master ветки (её зовут 
upstreem) необходимо всегда держать стабильно работающий вариант.
